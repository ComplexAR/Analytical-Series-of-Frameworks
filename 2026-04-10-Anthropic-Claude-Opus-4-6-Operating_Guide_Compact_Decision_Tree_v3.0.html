Index §2.2 Group-to-Hiddens linkage — what each group typically generates §2.2 ▶
G1 Framing (7 frameworks): Wrong boundary, wrong unit of analysis, wrong time horizon, misclassification of situation type, spatial blindness. Symptoms: "solving the wrong problem"; surprises at edges; location-dependent effects missed.

G2 Mechanism (10 frameworks): Hidden interactions, tight coupling, feedback delays, uncontrolled dependencies, emergent behaviour, network topology effects, institutional inertia, technological opacity, coordination failures. Symptoms: cascades, unexplained oscillation, fragile equilibria, emergent surprises, infrastructure lock-in.

G3 Agency (10 frameworks): Incentive suppression, power-filtered reporting, responsibility diffusion, strategic concealment, interest divergence, emotional distortion, unresolved conflict, cognitive bias. Symptoms: "everyone knew but nobody acted"; silence; denial; blame games; irrational persistence; escalation spirals.

G4 Meaning (8 frameworks): Narrative closure, legitimacy theatre, norm constraints on speaking, communications bottlenecks, ethical blind spots, trust erosion or misplaced trust. Symptoms: overconfidence; story dominance; ritual compliance; moral disengagement; trust failures.

G5 Epistemics (8 frameworks): Weak evidence, model overreach, untested assumptions, hindsight bias, diagnostic mono-culture, metric gaming, knowledge silos. Symptoms: conflicting accounts; false certainty; repeated investigative failure; metric-goal displacement.

G6 Action & Control (7 frameworks): Control drift, governance gaps, risk appetite mismatch, brittle decisions, learning not institutionalised, implementation gaps, scenario blindness. Symptoms: recurrence; "fix" without effect; slow degradation; plan-reality divergence; unexamined futures.

Gate Speed gate — select run mode §D.6 §4.1 ▶
Decision: What are the time constraints and decision stakes?

§D.6.1 Mode definitions:
• Rapid Run Mode (60–180 min): stabilise understanding, prioritise next steps. Uses §5.1 protocol. Light coverage with targeted depth.
• Investigative Run Mode (days–weeks): full explanation, accountability, recurrence prevention. Uses §5.2 or §5.3 protocol. Broad coverage with selective depth.

§D.6.7 Both modes exist because decisions have different urgency/stakes profiles. Rapid is not "worse" — it is fit-for-purpose under time pressure.
§D.6.4 No-Early-Exclusion Rule: even in Rapid, no group is excluded from at least a light scan. With 50 frameworks, the light scan across all 6 groups is broader — each group now has 7–10 frameworks providing more analytical surface area.
Rapid Run Mode (60–180 min)
Protocol §5.1 Rapid Protocol — 7-step sequence §5.1 ▶
Rapid
Step 1: Case question + timebox → G1
Step 2: Tier 1 Hiddens scan (assume hiddens exist) → §7.1 G5
Step 3: Boundaries, time horizon, scale, spatial context → G1
Step 4: System structure + dependencies, network topology, emergent properties → G2
Step 5: Evidence quality + key gaps → G5
Step 6: Immediate risks + stabilisation → G6
Step 7: Decide depth + plan → G6

Gate at Step 7: Commit to Rapid closure, or escalate to Investigative Run Mode.
Artefacts: §6 minimum schemas apply (slimmed). F/I/A/U tagging required on all claims (§D.3).

Phase 2 — Routing and Depth Selection
▼
Gate Route — scenario features → group coverage + depth §4.3 §3.2 ▶
Both
§4.3.1 Decision tree (compact):

Branch A — Failure occurred?
• YES → Minimum: G1+G2+G5 DEEP
  A1: Evidence weak/contested? → Deepen G5 frameworks (Evidence, Uncertainties, Diagnosis, Failures, Metrics, Knowledge)
  A2: Coupling/cascade? → Deepen G2 (Systems, Relations, Processes, Networks, Emergence, Technology + Risk)
  A3: Incentives/power/concealment? → Add G3 DEEP (now includes Interests, Emotions, Conflict, Cognition)
  A4: Narratives contested/"too clean"? → Add G4 DEEP (now includes Ethics, Trust)
  A5: Recommendations change controls? → Add G6 DEEP (now includes Implementation, Scenarios)
  A6: Adversarial intent? → Full G1–G6 DEEP; Tier 3 Hiddens (§7.3)
  A7: Institutional/technological mediation? → Deepen G2 (Institutions, Technology, Coordination)
  A8: Trust failure or ethical breach? → Add G4 DEEP (Trust, Ethics, Legitimacy)

Branch B — No failure, high-stakes decision pending?
• YES → Minimum: G1+G2+G5+G6 DEEP
  B1: Multi-stakeholder conflict? → Add G3 DEEP (Conflict, Interests, Emotions now available)
  B2: Narrative/legitimacy disputes? → Add G4 DEEP
  B3: High novelty/tight time? → Deepen G2+G5 (Emergence, Technology particularly relevant)
  B4: Implementation risk or scenario uncertainty? → Deepen G6 (Implementation, Scenarios, Risk)

• NO to both → Low-stakes/exploratory: G1+G5 light + one domain group deep

§4.3.2 Output contract: Routing statement, minimum group coverage, Full Depth list + rationale, Light Depth list + rationale, escalation triggers.
§3.2 Framework-to-Group mapping tells you which of the 50 frameworks sit in each group. With expanded groups (G2 and G3 now have 10 frameworks each), routing decisions must consider which specific frameworks within a deepened group are highest-signal.
§4.2 Depth selection: run deeply where consequence is high, evidence is weak, incentives may distort, or coupling exists.
Phase 3 — Iterative Execution Loop
▼
Loop §5.0 Iterative investigation loop — the master cycle §5.0 ▶
Both
All three protocols (§5.1, §5.2, §5.3) execute within this single repeatable loop. The loop enforces register-first execution, repeated Hiddens scanning, evidence-gated confidence, and explicit residual unknowns.

The loop phases are: Initialise → Generate → Route → Execute → Test → Re-scan → Decide
§5.0.3 Iteration triggers force re-entry: contradictory evidence, "too clean" story, high-impact unknown, new artefacts, governance implications, emergent behaviour detected, trust breakdown, coordination failure.
§5.0.4 Stop conditions: timebox requires output AND evidence supports claims AND residual unknowns are explicit AND detection interfaces exist.
Execute Run Full Depth frameworks — extract, populate, discover §5.0.2 step 4 ▶
Both
For each framework on the Full Depth list:
a) Extract facts from scenario + artefacts; tag F/I/A/U (§D.3.2)
b) Populate framework core tables (Types / Dynamics / Interfaces)
c) Add/adjust candidate hiddens (mechanism + impact + detectability)
d) Add tests/probes; update Evidence Ledger

Tier 2 LLM Execution Extracts: For LLM-assisted execution, load the compact extract (~5,000–8,000 tokens) for each Full Depth framework rather than the full document (~25,000 tokens). Extracts contain routing triggers, core tables, interface specifications, and near-neighbour disambiguation.

§6 Standard Artefacts: Hiddens Register, Evidence Register, Hypothesis/Test Backlog, Group Coverage Matrix
§D.3 Prompt Discipline Rules PD-01 to PD-10 govern all outputs
Execute Run Light Depth frameworks — scan for high-signal candidates §5.0.2 step 5 ▶
Both
For each framework on the Light Depth list:
a) Run scan prompts; record only high-signal candidates and missing-information triggers
b) A Light framework may escalate to Full Depth if evidence is contested, consequence is high, or coupling is discovered

§D.6.4 No-Early-Exclusion Rule: Every group gets at least a light scan, even in Rapid Run Mode. Exclusion is a depth decision, not a coverage decision. With 50 frameworks, light scanning covers more analytical territory per group — particularly G2 (10 frameworks) and G3 (10 frameworks).
Hiddens §7.1–7.5 Hiddens discipline fires within execution §7 ▶
Both
§7.1 Tier 1 Six-Category Scan: Required early (step 2 of loop) and repeated pre-closure (step 7). Sweeps all 6 meta-categories (I Perceptual → VI Ontological). Assume hiddens exist.

§7.2 Tier 2 (Crosswalk Shortlist) Micro-Protocol: Structured deepening of top Hiddens candidates. For each shortlisted hidden — rate mechanism, reducibility, detectability, agency, consequence. Assign detection + remediation interfaces. Map interaction chains.

§7.3 Escalation to Tier 3: Triggers: high consequence, adversarial concealment, persistent disagreement, repeat failure, cascade suspicion. Tier 3 = full 30-type run + detection/remediation matrix.

§7.5 Targeted Scans (13 scans, 4 families): When Tier 1 symptoms point to a specific hiding mechanism, use the Triage Matrix (§7.5.3) to select one or more scans. Each scan routes through 3–5 frameworks in sequence. v3.0 extends scan framework pools — new frameworks are available as secondary routing options where their analytical lens applies.

Family A Social/organisational:
• Scan 1 (Suppression): Power → Incentives → Culture & Norms → Communications → Hiddens [+Interests, Conflict]
• Scan 5 (Legitimacy Theatre): Legitimacy → Governance → Culture & Norms → Responsibility → Hiddens [+Institutions, Ethics, Trust]
• Scan 8 (Narrative Capture): Narratives → Perspectives → Evidence → Culture & Norms → Hiddens [+Trust, Emotions, Cognition]
• Scan 11 (Belief Persistence): Beliefs → Values → Evidence → Diagnosis → Hiddens [+Cognition, Emotions, Ethics]

Family B Structural/systemic:
• Scan 3 (Positional Blindness): Perspectives → Boundaries → Scale → Dimensions → Hiddens [+Space and Place, Cognition]
• Scan 6 (Cascade Discovery): Systems → Relations → Causation → Boundaries → Hiddens [+Networks, Emergence, Technology]
• Scan 9 (Uncertainty Masking): Uncertainties → Evidence → Metrics → Decisions → Hiddens [+Scenarios, Cognition]
• Scan 12 (Competency Blindspot): Competencies → Agents → Resources → Processes → Hiddens [+Coordination, Technology]

Family C Temporal:
• Scan 2 (Drift): Time → Processes → Governance → Failures → Hiddens [+Institutions, Implementation]
• Scan 7 (Amnesia): Learning & Adaptation → Knowledge → Failures → Governance → Hiddens [+Institutions, Cognition]

Phase 4 — Quality Controls
▼
Quality §8 Quality controls + anti-theatre rule §8 §10 ▶
Both
§8.1 Checklist: Alternative explanations maintained? Evidence graded? Incentive/power audit done? Narrative audit done? Boundary regression tested? Drift detection applied? Escalation discipline followed? Trust dynamics assessed? Ethical dimensions considered? Institutional constraints checked?

§8.2 Anti-theatre rule: Every major claim must be backed by: at least one evidence entry (or explicit gap), an uncertainty statement, a hiddenness check, and an owner for follow-up.

§10 Operating Principles: Mandatory constraints that govern all work — visibility-first, proportionality, explicit residual unknowns, no premature closure.

§D.3.1 PD-08: Do not close prematurely. PD-09: Make residual hiddenness explicit. PD-10: Canonical source rule.

▼
Gate Closure gate — are stop conditions met? §5.0.4 §7.4 ▶
Both
§5.0.4 Stop conditions (ALL must hold):
1. Decision timebox requires an output
2. Evidence Ledger supports key claims at required confidence
3. Residual Unknowns are explicit, prioritised, and assigned owners/actions
4. Remediation plan includes detection interfaces for top residual hiddens

If NOT met: Output must be non-closure with explicit close-out requirements, or escalate to Tier 3 / full G1–G6.
Stop conditions MET → Bounded closure
