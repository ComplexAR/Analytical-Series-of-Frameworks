## Section 2: Operational Definition

The Competencies framework identifies what capabilities (knowledge, skill, judgment, adaptability, self-awareness) agents possess or lack; classifies competencies into six meta-categories and thirty core types; profiles each competency across four dimensions (Depth/Mastery, Breadth, Resilience, Transferability); maps three dynamic patterns (Degradation, Development, Collapse); and surfaces hidden competency gaps, credential-capability divergence, and pressure-performance failures that enable errors and undermine resilience. It populates the Hiddens Register, Evidence Ledger, and Hypothesis/Test Backlog with competency-level blindnesses and capability-building requirements, enabling downstream frameworks to design roles, development, and safeguards aligned with true capability rather than credentials or tenure.

## Section 5: Condensed Dynamics

| Dynamic pattern | Signature (what you observe) | Mechanism (why it happens) | Consequence |
|---|---|---|---|
| Degradation | Competency erodes over time; skills atrophy; knowledge becomes obsolete; pressure performance collapses. | Non-use, routine conditions, lack of challenge/feedback, tech obsolescence, cognitive decline, fatigue. | Performance failures; safety risks; innovation capacity lost; crisis response capability degraded. |
| Development | Competency grows through practice, feedback, learning, and challenge; capabilities expand; transfer improves. | Deliberate practice, feedback integration, challenge/stretch assignments, explicit learning, reflection, mentoring. | Capability reaches new levels; adaptability improves; resilience increases; innovation capacity grows. |
| Collapse | Competency suddenly fails under novel conditions, pressure, or stakes exceeding previous experience. | Conditions exceed training envelope; pressure overwhelms cognitive capacity; novel context requires transfer that fails. | Crisis performance failures; unexpected errors; decisions that create new harm; loss of trust. |

## Section 6: Compact Hiddens Cross-Check Prompts

### 6a. Scan Questions by Meta-Category (I–VI)

| Hiddens meta-category | Scan question (1 sentence max) |
|---|---|
| I (Perceptual) | Which competency gaps are invisible because credentials mask lack of current capability; which skill atrophy goes undetected? |
| II (Systemic) | How do credential/competency mismatches persist in institutions; where are incompetent gatekeepers or teachers? |
| III (Informational) | Which competency information is distorted by self-assessment bias; where is capability measurement gaming? |
| IV (Temporal) | Which competencies have degraded silently; which pressure-performance gaps exist but are only visible in crisis? |
| V (Relational) | Which capability gaps are hidden by hierarchy (high-status people assumed to be competent); where is incompetence protected? |
| VI (Ontological) | Which competencies are system-level but attributed to individuals; where does competency distribution reflect hidden biases? |

### 6b. Primary Hiding Mechanisms This Framework Detects

- Credential-Competency Divergence (certified but not capable)
- Competency Concentration (expertise in individuals; transfer failures when they leave)
- Pressure-Performance Collapse (competent in routine; failures under pressure/novel conditions)
- Skill Atrophy (competency degrades from non-use; undetected until needed)
- Self-Assessment Bias (overconfidence hiding real limits)
- Competency Asymmetries hidden by Hierarchy (incompetence protected by status)
- Transfer Failures (competency does not transfer to adjacent domains)

### 6c. Detection/Remediation Interface Pointers

| When this type of Hidden emerges | Invoke this framework | Interface type | Action |
|---|---|---|---|
| Credential-competency mismatch | Framework of Evidence; Framework of Causation | Capability testing; competency audit; performance data review | Test actual capability; compare to credential; identify gaps; redesign roles or training. |
| Competency concentration in individuals | Framework of Systems; Framework of Risk | Knowledge mapping; dependency analysis; transfer success audit | Document tacit knowledge; develop backup capability; cross-train; reduce single-point-of-failure risk. |
| Pressure-performance collapse | Framework of Risk; Framework of Decisions | Stress-test scenarios; crisis simulation; decision quality under pressure assessment | Train under pressure; design supports/safeguards; identify pressure-sensitive roles; monitor stress levels. |
| Skill atrophy undetected | Framework of Learning; Framework of Evidence | Capability refresher audit; skill decay timeline; recent-use history | Create refresher training schedule; rotate roles to maintain skill; monitor decay signals. |
| Hierarchy protecting incompetence | Framework of Power; Framework of Governance | 360 feedback; upward feedback channels; peer assessment; independent review | Create safe feedback channels; separate measurement from hierarchy; ensure accountability for competency. |

## Section 7: LLM Execution Protocol

### 7a. Light Depth Execution (Scan-level)

1. List focal roles and critical decision-points; identify agents in those roles.
2. For each focal role, quickly assess: (a) what competencies are required, (b) which agents have those competencies (from credentials or known track record).
3. Identify obvious gaps (role requires expertise agent lacks) and obvious concentrations (expertise in one person).
4. Assess two dimensions: Credential-Competency Alignment (does credential match performance?) and Pressure-Resilience (will competency hold under pressure?).
5. Run Tier 1 Hiddens scan; shortlist top 2–3 competency gaps; flag for Tier 2 if high-consequence.

### 7b. Full Depth Execution (Complete framework run)

1. Map all focal roles and critical functions; create competency register (six meta-categories × 5 types per role).
2. For each focal role, assess required competencies across six meta-categories; identify criticality and depth.
3. For each focal agent, profile existing competencies across six meta-categories with evidence (credentials, track record, testing, peer feedback).
4. Identify gaps: required competency not possessed; profile depth/transfer/resilience.
5. Assess four dimensions for each competency: Depth/Mastery (surface/proficient/expert), Breadth (narrow/domain-specific/cross-domain), Resilience (routine-only/moderate-pressure/high-pressure), Transferability (non-transfer/adjacent-transfer/broad-transfer).
6. Map three dynamic patterns (Degradation, Development, Collapse); identify which are active for each agent; assess risk.
7. Run Tier 1 + Tier 2 Hiddens scan: credential mismatch, concentration risk, pressure-collapse risk, atrophy risk, hierarchy-protected incompetence; assign remediation (training, cross-training, safeguards, testing, rotation).
8. Design competency development and risk mitigation; document residual Unknowns; apply Escalation Rule; generate closure artefacts.

### 7c. Register Updates and Gate Question

| Register | Update type | Trigger |
|---|---|---|
| Hiddens Register | Add shortlist of competency-specific Hiddens with meta-category, gap, consequence, pressure-risk, mitigation lever. | Tier 1 scan completion; escalate any high-consequence Hidden to Tier 2. |
| Evidence Ledger | Record competency assessments, credential-performance alignment, gap evidence, testing results, pressure-performance data. | After competency profiling; note gaps (e.g., pressure-performance data missing, transfer success unknown). |
| Hypothesis/Test Backlog | Add hypotheses about competency gaps, pressure-collapse risk, transfer failures; propose tests (capability testing, pressure simulation, feedback audit). | Identified Unknowns and residual Hiddens; escalation decisions. |
| Information Requests | Request credentials, performance data, test results, 360 feedback, training records, incident data showing competency performance. | To reduce Unknowns and validate Hiddens detection; coordinate with Evidence and Learning frameworks. |

**Gate Question (pre-closure check):** Have we assessed required competencies across six meta-categories for all focal roles, profiled existing agent competencies across six dimensions, identified gaps and concentration risks, analysed three dynamic patterns, run a Tier 1 Hiddens scan, and escalated any high-consequence competency gaps or pressure-collapse risks to Tier 2/3 or downstream frameworks (Training, Risk, Governance)?
