## Section 2: Operational Definition

The Resources framework inventories and characterizes the material, informational, financial, human, and institutional resources that enable or constrain action in a scenario; classifies resource types across six meta-categories (Material, Informational, Financial, Human, Institutional, Relational); profiles resources along five dimensions (scarcity, accessibility, fungibility, renewal, control); analyzes resource dynamics (depletion, distribution, concentration, allocation); and surfaces resource-level Hiddens (hidden scarcity, resource fragility, control asymmetries, allocation blind spots). It populates the Resource Register, Evidence Ledger, and Capability/Constraint map with resource-level strategic analysis, enabling identification of resource bottlenecks and resource-dependent intervention feasibility.

## Section 5: Condensed Dynamics

| Dynamic pattern | Signature (what you observe) | Mechanism (why it happens) | Consequence |
|---|---|---|---|
| Hidden Scarcity | Resource appears available until crunch arrives; shortage emerges suddenly; capacity was illusory. | Scarcity masked by slack or reserve; usage grows until buffer exhausted; depletion rate exceeds renewal; monitoring insufficient. | Crisis; rationing; conflict over allocation; intervention stalled. |
| Resource Concentration & Monopoly | Critical resource controlled by single actor; concentration drives power asymmetry; alternatives unavailable. | Specialization; economies of scale; first-mover advantage; switching costs; alternatives not maintained. | Power imbalance; dependency; exploitation risk; vulnerability to action actor decisions. |
| Resource Fragmentation & Inefficiency | Needed resources scattered across many holders; coordination required; overhead high; efficiency lost. | Specialization; institutional siloes; governance fragmentation; no integration mechanism. | Coordination failure; inefficiency; missed opportunities; conflict over resource integration. |
| Resource Depletion & Renewal Gap | Resource depletes faster than renewal rate; trajectory unsustainable; crisis unavoidable unless renewal increases or depletion decreases. | Renewable resource treated as infinite; renewal rate underestimated; extraction rate unsustainable; discount future heavily. | Collapse; resource exhaustion; crisis; shift to alternative resource or activity redesign necessary. |
| Allocation Failure & Misallocation | Available resources poorly allocated; some uses unproductive or wasteful; better uses starved; inefficiency systemic. | Allocation mechanism ineffective (political, market, hierarchical); information gaps; incentive misalignment; power imbalance. | Ineffectiveness; missed opportunity; resource waste; low morale in starved areas. |
| Hidden Constraint & Bottleneck | Critical resource appears abundant but constrains activity; bottleneck not obvious until capacity tested. | Resource is latent constraint (e.g., expertise, authority, coordination bandwidth); appears plentiful in normal operation; constraint emerges under load. | Scaling failure; throughput limited; surprise when constraint discovered; redesign needed. |
| Fungibility Loss & Specialized Resources | Resource specialized for one use; alternative uses unavailable; if primary use disappears, resource becomes worthless. | Optimization for single purpose; alternatives not maintained; skills/tools narrow; switching costs high. | Stranded resources; sunk cost; poor resilience; crisis if primary use disappears. |

### 6a. Scan Questions by Meta-Category (I–VI)

| Hiddens meta-category | Scan question (1 sentence max) |
|---|---|
| I (Perceptual) | What resources are salient vs invisible from common positions; what resource constraints operate below attention? |
| II (Systemic) | How do resources couple with institutions and incentives; what resource allocation failures emerge from system structure? |
| III (Informational) | What information about resource availability is available vs hidden; what resource accounting is missing? |
| IV (Temporal) | What resources are depleting at unsustainable rates; what renewal gaps will emerge; what resource shifts are imminent? |
| V (Relational) | Who controls key resources; whose resource access is constrained; what power asymmetries are rooted in resource control? |
| VI (Ontological) | What resource types are not yet recognized; what emergent resource constraints are unnameable? |

### 6b. Primary Hiding Mechanisms This Framework Detects

- Resource Invisibility (abundant resources until crunch; scarcity latent)
- Control Concealment (resource monopoly hidden by availability illusion)
- Allocation Blindness (misallocation hidden by overall resource adequacy)
- Depletion Lag (renewable resource depletion invisible until irreversible threshold crossed)
- Fragmentation Inefficiency (distributed resources fail to coordinate; inefficiency attributed to other causes)
- Constraint Latency (resource bottleneck emerges under load; not visible in normal conditions)
- Fungibility Blindness (specialized resources appear worthless if primary use disappears)

### 6c. Detection/Remediation Interface Pointers

| When this type of Hidden emerges | Invoke this framework | Interface type | Action |
|---|---|---|---|
| Resource crunch or bottleneck | Framework of Evidence; Framework of Systems | Resource capacity audit; utilization tracking; constraint identification | Monitor resource utilization; identify bottlenecks; plan alternatives. |
| Resource concentration or monopoly | Framework of Power; Framework of Governance | Control mapping; concentration analysis; diversification design | Expose concentration; build alternatives; distribute control. |
| Depletion or sustainability threat | Framework of Time; Framework of Risk | Depletion rate tracking; renewal rate analysis; sustainability assessment | Measure extraction vs renewal; project trajectory; design sustainable rates. |

## Section 7: LLM Execution Protocol

### 7a. Light Depth Execution (Scan-level)

1. Identify focal activity and critical resources: list resources (I–VI) required for activity to proceed; note single points of failure.
2. For each critical resource, assess: Scarcity (abundant/adequate/tight), Control (distributed/concentrated), Accessibility (easy/difficult/unavailable).
3. Run Tier 1 Hiddens scan: touch all six meta-categories; flag obvious resource bottlenecks, concentration, or depletion risks.
4. Shortlist 2–3 highest-consequence resource risks (bottleneck, concentration, depletion); note if resource constraint could stop activity; flag for escalation.

### 7b. Full Depth Execution (Complete framework run)

1. Resource inventory: identify all resources (I–VI) required for focal activity; for each, record availability, scarcity, control, accessibility, renewal.
2. Resource profile: for each critical resource, assess across dimensions (scarcity, accessibility, fungibility, renewal, control); note asymmetries.
3. Depletion analysis: for each renewable resource, estimate depletion rate and renewal rate; project trajectory; identify unsustainability. 
4. Allocation map: trace how resources are allocated; identify allocation mechanism (political, market, hierarchical); assess fairness and efficiency.
5. Concentration analysis: identify which resources are concentrated; assess who controls them; estimate power asymmetry and dependency risk.
6. Constraint testing: identify latent constraints by running scenarios; test whether resources adequate under stress; identify bottlenecks.
7. Run Tier 1 + Tier 2 Hiddens scan: for each high-consequence resource Hidden, identify mechanism, detectability, consequence; propose resource tests.

### 7c. Register Updates and Gate Question

| Register | Update type | Trigger |
|---|---|---|
| Hiddens Register | Add resource-level Hiddens with meta-category tag (Material, Informational, Financial, Human, Institutional, Relational); mechanism; consequence. | After Tier 1 scan; escalate bottlenecks and concentration risks. |
| Evidence Ledger | Record resource inventories, scarcity/control profiles, allocation patterns, depletion rates; note measurement gaps. | After resource inventory and profile analysis. |
| Hypothesis/Test Backlog | Add hypotheses about resource constraints, depletion trajectories, concentration impacts; propose resource-stress tests. | Identified resource Unknowns; escalation decisions. |
| Information Requests | Request resource inventory data; request allocation mechanism documentation; request depletion/renewal rate data. | To reduce resource Unknowns; coordinate with Evidence and Systems frameworks. |

**Gate Question (pre-closure check):** Have we inventoried focal resources, profiled critical resources across dimensions, identified bottlenecks and concentrations, assessed depletion and allocation, run a Tier 1 Hiddens scan, and designed resource-aware interventions?
