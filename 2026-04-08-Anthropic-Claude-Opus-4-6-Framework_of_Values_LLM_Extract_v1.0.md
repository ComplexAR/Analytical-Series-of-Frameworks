## Section 2: Operational Definition

The Values framework identifies what matters in a scenario—classifying values into six meta-categories and thirty core types, profiling them across five dimensions (Scope, Weight, Conditionality, Source, Comparability), and mapping their interactions through five dynamics (Conflict, Reinforcement, Hierarchy, Transformation, Realisation). It surfaces four systematic sources of value visibility failure (Value Blindness, Value Monism, Category Neglect, Fixity Assumption) through tiered Hiddens scanning and populates the Hiddens Register, Evidence Ledger, Hypothesis/Test Backlog, and Information Requests with value-specific blindnesses and governance requirements, enabling downstream frameworks to design decision criteria, governance rules, and value-conflict remediation.

## Section 4: Condensed Core Taxonomy

| Meta-category | Core types (names only) | Diagnostic question (1 sentence) |
|---|---|---|
| I. Foundational Values | Intrinsic, Instrumental, Constitutive, Final, Contributory | What makes something valuable in itself, and on what basis do we pursue means toward valued ends? |
| II. Ethical Values | Moral, Justice, Virtue, Care, Dignity | What should we do, and how should persons be treated? |
| III. Epistemic Values | Truth, Justification, Understanding, Coherence, Explanatory Power | What makes beliefs, claims, and knowledge good? |
| IV. Aesthetic Values | Beauty, Elegance, Sublimity, Expression, Craft | What makes things beautiful, expressive, elegant, or sublime? |
| V. Practical Values | Wellbeing, Prudential, Efficiency, Security, Freedom | What supports wellbeing and flourishing in concrete life? |
| VI. Social Values | Cultural, Solidarity, Belonging, Sacred, Ecological | What makes collective life and community valuable? |

## Section 5: Condensed Dynamics

| Dynamic pattern | Signature (what you observe) | Mechanism (why it happens) | Consequence |
|---|---|---|---|
| Conflict | Values coming into tension; pursuing one sacrifices another; trade-off pressure; stakeholder misalignment. | Incompatible ends; resource scarcity; different stakeholder priorities; incommensurable value types. | Value stakes become political; decisions become contested; legitimate grievance and backlash; delayed or failed implementation. |
| Reinforcement | Values supporting and strengthening each other; synergy; mutual alignment across stakeholder groups. | Complementary ends; shared stakeholder interests; values that depend on each other (e.g., truth and understanding). | Easier adoption; reduced resistance; momentum; mutually justifying decisions; stronger collective commitment. |
| Hierarchy | Values ordered in relations of priority; some constrain or outrank others; absolute constraints; sacred values. | Principled ordering; power structures embedding preferences; inviolable/non-tradeable values marked as sacred. | Clear decision criteria; sacred violations become detect-able; constraints protect valued goods; potential brittleness when ordering is challenged. |
| Transformation | Values changing over time in weight, scope, meaning, or content; value drift undetected; generational shifts; cultural change. | Slow adaptation; organisational incentive drift; generational cohort shifts; external regulatory or market shifts; silenced value concerns. | Stale mission statements; sudden conflicts; performance misaligned with stated values; repeated "why are we doing this?" questions; institutional legitimacy erosion. |
| Realisation | Gap between what is valued and what is actual; value is declared but not pursued; instantiation requires work and trade-offs. | Constraints prevent instantiation; other values block pursuit; actual incentives misaligned with stated values; resource limits or resistance. | Value-outcome misalignment visible; hypocrisy and trust erosion; motivation gaps between espoused and operative values. |

## Section 6: Compact Hiddens Cross-Check Prompts

### 6a. Scan Questions by Meta-Category (I–VI)

| Hiddens meta-category | Scan question (1 sentence max) |
|---|---|
| I (Perceptual) | What values are implicit and never made explicit; whose values are invisible to leadership; what would stakeholders find unacceptable that no one has named? |
| II (Systemic) | What values is the system actually optimising for (via metrics, incentives, structure) versus claiming; are single-metric or single-objective assumptions hiding value trade-offs? |
| III (Informational) | What value perspectives are missing from reporting and briefing channels; whose values are filtered out in communication flows; are value concerns suppressed in sanitised reports? |
| IV (Temporal) | What values have drifted silently from stated mission; are there generational, market, or cultural shifts in what actually matters; how long since values were re-elicited? |
| V (Relational) | How do position, power, and identity shape what can be valued and voiced; are certain stakeholder values legitimately absent or relationally silenced; who cannot speak? |
| VI (Ontological) | Are the six meta-categories adequate to what matters in this scenario; are there persistent value anomalies or exceptions; do new forms of value need recognition? |

### 6b. Primary Hiding Mechanisms This Framework Detects

- Value Blindness (failing to see operative values; implicit values never named or examined)
- Value Monism (reducing all values to a single type—profit, efficiency, safety, growth—masking trade-offs and incommensurability)
- Category Neglect (ignoring entire meta-categories of values; aesthetic, ecological, or social values treated as optional luxuries)
- Fixity Assumption (treating dynamic values as fixed; outdated codes and missions; ignoring value drift and cultural change)
- Value Instrumentalisation (values invoked rhetorically to block critique rather than guide decision-making; performance masking)
- Value-Outcome Misalignment (declared values contradicted by incentives and outcomes; hypocrisy visible only in failure)

### 6c. Detection/Remediation Interface Pointers

| When this type of Hidden emerges | Invoke this framework | Interface type | Action |
|---|---|---|---|
| Value Blindness in stakeholder elicitation | Framework of Perspectives; Framework of Hiddens (Tier 2) | Structured interviews; sorting exercises; hypothetical scenarios; observational value inference | Expand stakeholder set; require explicit value register with provenance; protect space for uncomfortable values; incentive analysis |
| Value Monism in metrics/incentives | Framework of Governance; Framework of Evidence | Objective function audit; proxy drift analysis; gaming vulnerability assessment | Enforce plural criteria; separate non-tradeable constraints; mark incommensurables explicitly; require trade-off rationale statements |
| Category Neglect (systematic omission) | Framework of Perspectives; Framework of Processes | Six-category audit; role-based blinder assessment; team composition review | Mandatory cross-category scan; rotate perspectives; add domain experts and affected parties; track consistent devaluation patterns |
| Value Drift and Fixity (temporal blindness) | Framework of Dynamics; Framework of Institutions | Historical baseline review; value re-elicitation tracking; drift indicator monitoring | Establish re-elicitation cadence (quarterly minimum); include transformation scenarios; monitor generational/market/regulatory shifts; flag staleness |
| Value-Outcome Misalignment | Framework of Evidence; Framework of Responsibility | Incentive-value alignment check; stated vs operative value analysis; outcome attribution analysis | Redesign incentives to match stated values; establish accountability for value delivery; close the gap between aspiration and actual practice |

## Section 7: LLM Execution Protocol

### 7a. Light Depth Execution (Scan-level)

1. List all stakeholder groups and elicit implicit values (what matters to each; what would be unacceptable; unstated priorities); map stakeholder value profiles.
2. Scan all six meta-categories (I–VI) and assign each identified value to one or more types; note if a category is entirely absent or devalued.
3. For each value identified, assess at least two dimensions (Scope: who holds it? Weight: how much does it matter? Source: is it espoused, operative, or hidden?); flag obvious misalignments.
4. Run Tier 1 Hiddens scan: touch all four value-specific hiding mechanisms lightly (Blindness, Monism, Category Neglect, Fixity); flag obvious perceptual, systemic, or temporal blindness.
5. Shortlist the 2–3 highest-consequence value conflicts or drifts; note escalation flag if high-consequence and high-uncertainty.

### 7b. Full Depth Execution (Complete framework run)

1. Elicit stakeholder values through multi-method inquiry (interviews, observation, incentive analysis, hypothetical scenario probing); create detailed value profile for each stakeholder group.
2. Classify all values across six meta-categories and thirty types; note domain-specific refinements or new types if needed (see Section 3.7 of parent MRT).
3. Profile all identified values across five dimensions (Scope, Weight, Conditionality, Source, Comparability); assess concordance or divergence across stakeholders; record espoused vs operative value gaps.
4. Map all five dynamics patterns: for each primary value, identify which dynamics are active (nascent, present, high-risk); document value conflicts, reinforcements, hierarchies, transformation signals, and realisation gaps.
5. Run three interface analyses: Justification (how values justify decisions), Evaluation (how values serve as judgment criteria), Motivation (how values enable/block action); assess sufficiency of each interface.
6. Run Tier 1 + Tier 2 Hiddens scan (structured deepening): for each high-relevance hiding mechanism (e.g., Value Blindness, Monism, Fixity), assess how it manifests across all six Hiddens meta-categories; identify detection tests and remediation levers.
7. Document unresolved value disputes and unknowns; apply Escalation Rule if any Unknown could change decisions or accountability; populate Residual Unknowns register and closure artefacts.

### 7c. Register Updates and Gate Question

| Register | Update type | Trigger |
|---|---|---|
| Hiddens Register | Add value-specific Hiddens with hiding mechanism tag (Blindness/Monism/Category Neglect/Fixity), meta-category manifestation (I–VI), detectability, consequence, stakeholder impact. | Tier 1 scan completion; escalate any high-consequence/high-uncertainty Hidden to Tier 2 and engage remediation design. |
| Evidence Ledger | Record value type classifications, dimensional profiles, stakeholder concordance/divergence, evidence quality on espoused vs operative values, provenance of value claims. | After stakeholder elicitation and dimensional profiling; note gaps (missing stakeholder voice, untested value claims, incentive analysis pending). |
| Hypothesis/Test Backlog | Add hypotheses about value conflicts, value drift, category neglect, and value-outcome misalignment; propose tests (value re-elicitation, incentive audit, outcome attribution analysis). | Identified Unknowns and residual Hiddens; escalation decisions; value transformation scenarios. |
| Information Requests | Request stakeholder interviews; incentive structure documentation; historical value statements and mission drift analysis; outcome data disaggregated by value impact. | To reduce Unknowns on stakeholder values and detect hidden value-outcome misalignment; coordinate with Evidence and Governance frameworks. |

**Gate Question (pre-closure check):** Have we identified and classified values across all six meta-categories, assessed at least two dimensions per primary value, detected obvious value conflicts and stake disparities, run a Tier 1 Hiddens scan across all four value-specific mechanisms and all six Hiddens meta-categories, and escalated any high-consequence value conflicts or unknowns to appropriate frameworks (Hiddens, Evidence, Governance, Decisions)?

## Section 8: Cross-Framework Routing Pointers

### 8a. Upstream Inputs (frameworks commonly feeding into this one)

- Situations Context Classification (what situation type makes which values salient?)
- Boundaries (what jurisdictional/regulatory constraints affect permissible values?)
- Scale & Time (what are the relevant time horizons for value stability and drift?)
- Agents & Personas (who are the stakeholders; what are their value contexts and identities?)
- Evidence (what evidence exists about stakeholder values, value-outcome alignment, contested value claims?)
- Power & Institutions (who controls whether values are pursued; how do institutions enable or enforce value priorities?)

### 8b. Downstream Handoffs (frameworks commonly consuming this framework's outputs)

- Framework of Hiddens (value-specific Hiddens detection and remediation design; visibility failure mapping)
- Framework of Governance (governance design to enforce value priorities, resolve conflicts, protect sacred values; accountability for value delivery)
- Framework of Decisions (decision criteria, constraint sets, trade-off rationale; value-informed choice architecture)
- Framework of Interventions (intervention design that respects value constraints and sacred values; redesign for value-outcome alignment)
- Framework of Risk (risk register including value conflicts as source of cascade risk; value-drift scenarios)
- Framework of Responsibility (accountability mapping for value delivery and value-outcome gaps; legitimacy assessment)

### 8c. Targeted Scans (OG §7.5 scans that invoke this framework)

| Targeted Scan | Role of this framework |
|---|---|

| Scan 2: Value blindness and implicit assumption visibility | Surface implicit values held by powerful stakeholders; surface operative values hidden by rhetorical claims; detect value conflicts suppressed as resolved. |
| Scan 4: Single-metric and value monism detection | Detect which values are being optimised (via metrics and objectives) versus ignored; assess trade-offs being hidden by single-number domination. |
| Scan 7: Value drift and temporal blindness detection | Identify values that have drifted from initial commitments; assess generational or cultural shifts undetected by organisations; flag stale mission statements. |
