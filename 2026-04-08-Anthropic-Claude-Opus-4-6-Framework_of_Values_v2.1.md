# Introduction

## What is Values?

Values are the structures of **mattering**: they specify what makes outcomes desirable or undesirable, actions permissible or impermissible, and beliefs acceptable or unacceptable. Whether understood as objective features of the world, as intersubjective social facts, or as subjective commitments, values shape what agents notice, what they choose, and what they can justify. In complex scenarios, value stakes are not always visible. A policy can achieve targets while eroding legitimacy. An intervention can succeed mechanically while violating dignity or destroying trust. A model can optimise the wrong thing because normative choices are buried in its objective function. This framework provides a disciplined vocabulary for identifying value stakes, classifying them systematically, profiling how they apply to stakeholders and contexts, and translating them into decision criteria and governance rules that prevent value blindness and value conflicts from generating repeated failures.

## Why does Values matter for Hiddens work?

Values-based hiding mechanisms are pervasive in complex organisations and systems. Implicit values remain invisible until they collide with explicit values in a crisis. Value conflicts are suppressed—declared resolved when they are merely postponed. Single-metric optimisation entraps organisations into pursuing one value at the expense of others, only discovering the cost when sacred values are violated. Value drift—the slow erosion of what an organisation actually values versus what it claims to value—persists undetected until cascading failures reveal the gap. This framework surfaces several families of Hiddens: **Value Blindness (Hidden-1)** operates when operative values are never named or examined; **Value Monism (Hidden-2)** operates when all values collapse into a single scale (profit, efficiency, safety); **Category Neglect (Hidden-3)** operates when entire value domains (aesthetic, ecological, social) are treated as optional luxuries; **Value Instrumentalisation (Hidden-11)** operates when values are invoked rhetorically to block critique rather than to guide decision-making. Without this framework, these Hiddens remain latent: invisible until they generate failure, regret, or cascade effects.

## What does this framework produce?

The framework operationalises value analysis through six core elements: (1) **A taxonomy of 30 value types** organised into six meta-categories that recur across contexts, allowing rapid initial classification and comparison; (2) **Five cross-cutting dimensions** (Scope, Weight, Conditionality, Source, Comparability) that profile any value and explain why different stakeholders classify the same value differently; (3) **Dynamics patterns** (Conflict, Reinforcement, Hierarchy, Transformation, Realisation) that describe how values interact, enabling detection of value drifts and early warning of normative shifts; (4) **Three interface types** (Justification, Evaluation, Motivation) that make visible where and how values connect to reasoning, judgment, and action; (5) **Hiddens source analysis and tiered Hiddens scans** that surface the four systematic sources of value visibility failure (Value Blindness, Value Monism, Category Neglect, Fixity Assumption) and the six categories of hiding mechanisms (Perceptual, Systemic, Informational, Temporal, Relational, Ontological); (6) **A six-step application protocol** that moves from situation clarification through meta-category scanning, type mapping, dimensional profiling, interface mapping, Hiddens checking, and integration handoff. The framework populates standard registers (Hiddens Register, Evidence Ledger, Hypothesis/Test Backlog, Information Requests) that integrate with the broader Analytical Series investigation workflow.

## How to use this framework

Invoke this framework early and repeatedly whenever normative disagreement is material to the decision or outcome. It is most valuable at scenario intake (to establish what values are at stake and where conflicts may arise), during major decision gates (to validate whether value priorities remain stable or have drifted), and when interventions fail to produce expected effects (often a signal of value misalignment or value cascade). The framework works in both Rapid Run Mode (light-touch value elicitation and conflict spotting) and Investigative Run Mode (full dimensional profiling, complete Hiddens scanning, multi-stakeholder value synthesis). Default execution is Light Depth Framework Execution: scan all six meta-categories, identify top priority values, profile one or two dimensions, run a Tier 1 Hiddens scan. Escalate to Full Depth when consequence is high, evidence of value conflicts is strong, or you are approaching a governance threshold. The framework's companion prompt discipline rules (OG v2.5, §D.3) require tagging all value claims as Fact, Interpretation, Assumption, or Unknown, and explicitly recording disagreements, uncertainties, and residual unknowns rather than collapsing them into false consensus. For LLM execution, see §1.6 for the complete LLM integration specification, standard registers, and copy-ready run prompts.

# 1. Core Premise & Position in the Analytical Series

## 1.1 Core Premise (narrative)

Values are the structures of **mattering**. They are what make outcomes desirable or undesirable, actions permissible or impermissible, and beliefs acceptable or unacceptable. Whether treated as objective features of the world, as intersubjective social facts, or as subjective commitments, values shape what agents notice, what they choose, and what they can justify.

In practice, value failures often look like "technical" failures: a model optimises the wrong thing; a policy achieves targets while eroding legitimacy; an intervention succeeds mechanically while violating dignity or destroying trust. Conversely, many apparently technical failures conceal normative errors: disagreements about what counts as success, what matters more, what trade-offs are acceptable. The Framework of Values provides a disciplined vocabulary for identifying value stakes, classifying them, profiling how they apply, and translating them into decision criteria and governance rules.

This framework produces:
- **A taxonomy of 30 value types** organised into six irreducible meta-categories (Foundational, Ethical, Epistemic, Aesthetic, Practical, Social)
- **Five dimensions for profiling any value** (Scope, Weight, Conditionality, Source, Comparability) that explain why the same value can be held differently across stakeholders and contexts
- **Value dynamics patterns** (Conflict, Reinforcement, Hierarchy, Transformation, Realisation) that describe how values interact and change
- **Interface mapping** that shows where values connect to reasoning (justification), judgment (evaluation), and action (motivation)
- **Hiddens source analysis** that surfaces value-based visibility failures and proposes remediation
- **An application protocol** that translates value analysis into decision criteria, evaluation rubrics, escalation rules, and governance assignments

## 1.3 Definitions, Scope, and Non-Goals

- **Value (operational)**: A normative salience that provides reasons, standards, or motivation—something that matters, for someone, in some way.
- **In scope**:
  - Six meta-categories and thirty core value types (baseline taxonomy)
  - Five dimensions for profiling any value (Scope, Weight, Conditionality, Source, Comparability)
  - Five value dynamics (Conflict, Reinforcement, Hierarchy, Transformation, Realisation)
  - Three value interface types (Justification, Evaluation, Motivation)
  - Hiddens source analysis + tiered Hiddens cross-check
  - Application protocol and deliverables
  - Local extensions and contextual emphasis (with mapping constraints)
- **Out of scope**:
  - A complete ethical theory (the framework supports plural normative theories rather than choosing one)
  - Full measurement methodology for values (e.g., cost–benefit, multi-criteria decision analysis); the framework clarifies when measurement is appropriate or illegitimate
  - Full catalogue of interventions and governance mechanisms (use Interventions, Governance, Responsibility, Legitimacy)
- **Common confusions**:
  - "Values are subjective preferences" — No; values provide normative reasons and structure choice, not merely individual taste
  - "All values are comparable on one scale" — No; some values are incommensurable and cannot be traded off
  - "Values are fixed" — No; values drift over time, contexts, and generations; periodic re-elicitation is required
  - "Values are private/internal" — No; values are embedded in institutions, incentives, policies, and public commitments; they can be identified and mapped

- Stakeholders hold materially different views on what success/harm/fairness means (not just factual disagreement)
- Analysis identifies "best solution" that will face unexpected resistance (signals value misalignment)
- Prior interventions or policies failed despite being technically sound (likely value constraint)
- Normative disagreement is material to decision (value stakes are outcome-shaping)
- Sacred values, non-tradeable constraints, or incommensurable trade-offs are suspected
- Value conflict is suppressed, implicit, or driving organisational silence or backlash

- Governance must manage value disputes or allocation trade-offs (pluralism rather than consensus assumed)

## 2.2 Meta-Category Descriptions (recommended)

### I. Foundational Values
Values that define the basic architecture of valuing: intrinsic worth, ends and means, and how parts contribute to wholes. What makes something valuable in itself, and on what basis do we pursue means toward valued ends?

### II. Ethical Values
Values governing right action and moral treatment: duties, rights, fairness, care, dignity, and character. What do we owe to each other, and how should persons be treated?

### III. Epistemic Values
Values governing good belief and knowledge: truth, justification, coherence, understanding, and explanatory power. What makes knowledge claims and reasoning sound?

### IV. Aesthetic Values
Values governing beauty, form, and expressive experience: beauty, elegance, craft, expression, and sublimity. What makes experience meaningful through sensory, formal, and expressive dimensions?

### V. Practical Values
Values governing concrete flourishing: wellbeing, security, freedom, efficiency, and prudence. What enables lives to go well for those living them?

### VI. Social Values
Values governing collective life: culture, belonging, solidarity, diversity, ecological integrity, and sacredness. What makes community, shared meaning, and natural systems valuable?

# 3. Core Types Taxonomy

## 3.0 Canonical Rule & Deviations
- **Canonical baseline**: 6 meta-categories × 5 types = 30 core value types (invariant vocabulary for cross-domain analysis).
- **Allowed deviations**:
  - Local extensions (Section 3.7) that add types while preserving the baseline and documenting mappings.
  - Contextual emphasis (prioritising a subset) is allowed, but category omission should be treated as a potential hidden (Section 8).
- **Disallowed deviations** (without explicit justification and version control):
  - Deleting core types, renaming them to non-equivalent labels, or collapsing categories into a single scale by default.

## 3.1 Types (Category I: Foundational Values)
| Value Type | Description |
| :---- | :---- |
| **Intrinsic** | Value something has in itself, regardless of consequences or relations. Something is intrinsically valuable if it would be good even in isolation, valuable for its own sake. |
| **Instrumental** | Value something has as a means to other valuable ends. Something is instrumentally valuable if it produces, enables, or contributes to other things that are valuable. |
| **Constitutive** | Value something has as a necessary part of something larger that is valuable. Constitutive value differs from instrumental — the part doesn't just cause the whole but partially comprises it. |
| **Final** | Value as an end or goal of action. Final values are what we ultimately aim at — they justify instrumental pursuits but are not themselves pursued for the sake of something else. |
| **Contributory** | Value something has by contributing to an overall valuable whole. Things can have positive or negative contributory value depending on how they affect the total. |

## 3.2 Types (Category II: Ethical Values)
| Value Type | Description |
| :---- | :---- |
| **Moral** | Value concerning right and wrong action, moral obligation, and what we owe to others. The domain of duty, rights, and ethical requirement. |
| **Justice** | Value concerning fair distribution, proper treatment, and giving each their due. Justice encompasses procedural, distributive, and corrective dimensions. |
| **Virtue** | Value of character traits that constitute moral excellence — courage, honesty, compassion, integrity. Virtues are stable dispositions toward right action and feeling. |
| **Care** | Value of attending to relationships, needs, and vulnerabilities. Care ethics emphasises responsiveness, connection, and responsibility for particular others. |
| **Dignity** | Value of persons as ends in themselves, not merely means. Dignity grounds respect, autonomy, and the impermissibility of certain treatments regardless of consequences. |

## 3.3 Types (Category III: Epistemic Values)
| Value Type | Description |
| :---- | :---- |
| **Truth** | Value of beliefs, statements, and theories being true — corresponding to reality, getting things right. The fundamental epistemic value against which others are often measured. |
| **Justification** | Value of beliefs being well-supported by evidence and reasoning. Justified beliefs are held for good reasons, not merely true by accident. |
| **Understanding** | Value of grasping how things work, why they are as they are, and how they connect. Understanding goes beyond knowing facts to comprehending relationships and explanations. |
| **Coherence** | Value of beliefs fitting together consistently, forming an integrated system. Coherent belief systems avoid contradiction and support mutual intelligibility. |
| **Explanatory Power** | Value of theories and frameworks that explain much with little — unifying diverse phenomena under common principles. Parsimony, scope, and depth matter. |

## 3.4 Types (Category IV: Aesthetic Values)
| Value Type | Description |
| :---- | :---- |
| **Beauty** | Value of perceptual and formal properties that please, delight, or inspire. Beauty encompasses natural beauty, artistic beauty, and the beauty of abstract forms. |
| **Elegance** | Value of achieving much with little — simplicity, efficiency, and grace in form. Elegant solutions, proofs, and designs accomplish goals with minimal complexity. |
| **Sublimity** | Value of experiences that overwhelm or transcend ordinary perception — vastness, power, mystery. The sublime evokes awe, wonder, and a sense of human smallness. |
| **Expression** | Value of articulating inner states, meanings, or visions through external form. Expressive value concerns how well works convey what they aim to communicate. |
| **Craft** | Value of skilled making — technical excellence, mastery of materials, refined execution. Craft values the quality of how things are made, not just what is made. |

## 3.5 Types (Category V: Practical Values)
| Value Type | Description |
| :---- | :---- |
| **Wellbeing** | Value of lives going well for those living them — happiness, satisfaction, flourishing. Wellbeing encompasses hedonic, desire-satisfaction, and objective-list theories. |
| **Prudential** | Value of what serves one's own long-term interests. Prudential value concerns practical wisdom about one's own good, balancing present and future. |
| **Efficiency** | Value of achieving goals with minimal waste of resources — time, energy, materials. Efficiency concerns the ratio of outputs to inputs. |
| **Security** | Value of protection from threats, stability, and predictability. Security encompasses physical safety, economic security, and psychological security. |
| **Freedom** | Value of being able to act according to one's own will — absence of constraint, presence of options, capacity for self-determination. |

## 3.6 Types (Category VI: Social Values)
| Value Type | Description |
| :---- | :---- |
| **Cultural** | Value of shared meanings, practices, and traditions that constitute cultural identity. Cultural value concerns the goods internal to particular ways of life. |
| **Solidarity** | Value of mutual support, shared commitment, and standing together. Solidarity involves identification with others and willingness to bear costs for collective goods. |
| **Belonging** | Value of being part of a community, being recognised and included. Belonging concerns social connection, identity, and the human need for community. |
| **Sacred** | Value of what is set apart as holy, inviolable, or transcendent. Sacred values resist trade-offs and mark what is beyond ordinary valuation. |
| **Ecological** | Value of natural systems, biodiversity, and environmental integrity. Ecological value concerns the goods of nature independent of human use. |

## 3.7 Extending the Taxonomy

The 30-type taxonomy is designed as a **canonical baseline**, not a closed list. Extension is appropriate when analysis repeatedly encounters values that are:

- **Persistent** (not a one-off preference), and
- **Decision-relevant** (they change what is justified, evaluated, or pursued), and
- **Not well-captured** by any existing core type without distortion or forced re-labelling.

### Extension criteria
A proposed new value type should satisfy all of the following:

1. **Non-redundancy:** It is not merely a synonym for an existing type.
2. **Distinctive functional role:** It changes reasoning at one or more value interfaces (justification/evaluation/motivation).
3. **Dimensional profile:** It can be characterised across all five dimensions (Scope, Weight, Conditionality, Source, Comparability).
4. **Dynamic behaviour:** It exhibits recognisable patterns across the five dynamics (Conflict, Reinforcement, Hierarchy, Transformation, Realisation).
5. **Meta-category fit:** It either (a) fits clearly within one meta-category, or (b) justifies a documented "bridge" note if it systematically spans categories.
6. **Operational test:** It produces different analytic outputs than adjacent types in at least one real case (better predictions, clearer trade-offs, better governance).

### Extension workflow (recommended)
1. **Candidate capture:** Record the candidate value in the value register with examples and stakeholder provenance.
2. **Nearest-neighbour mapping:** Identify the 2–3 closest existing value types and document why they fail.
3. **Provisional definition:** Write a definition that distinguishes the type by role (not by rhetoric).
4. **Stress test:** Apply the candidate across two contrasting cases; check dimensional and dynamic consistency.
5. **Interface validation:** Show how the candidate changes at least one interface: a different reason, a different standard, or a different motivational lever.
6. **Governance decision:** Accept/reject as a local extension; if accepted, add to local taxonomy appendix with version control.
7. **Series alignment:** If promoted to the canonical series, update cross-framework mappings (Sections 7–9) and the glossary.

### Compatibility rule
Local extensions must preserve comparability with the series baseline by:
- Keeping the original 30 types intact (no deletion/renaming), and
- Documenting the mapping from the new type to the nearest baseline types and to the five dimensions/dynamics.

# 4. Dimensions
*How any value can be profiled for scope, priority, applicability, grounding, and trade-off structure.*

## 4.1 Dimensions Summary Table (mandatory)
| Dimension | Core question | What it captures | Representative value-positions (non-exhaustive) |
|---|---|---|---|
| 1. Scope | Who/what does the value apply to? | Domain and extent of applicability. | Universal; General; Contextual; Particular; Personal |
| 2. Weight | How important is the value relative to others? | Priority/strength in trade-offs and conflicts. | Absolute; Dominant; Significant; Moderate; Minimal |
| 3. Conditionality | Under what conditions does the value apply? | Whether applicability is unconditional, defeasible, or goal-dependent. | Unconditional; Prima facie; Conditional; Contingent; Hypothetical |
| 4. Source | Where does the value come from? | Ontological and social grounding of the value. | Objective; Intersubjective; Subjective; Constructed; Emergent |
| 5. Comparability | Can it be compared or traded off? | Commensuration, ranking, and incommensurability. | Commensurable; Comparable; Roughly comparable; Incomparable; Incommensurable |

## 4.2 Dimension Details (recommended)

### 4.2.1 Dimension 1: Scope
*Who or what the value applies to.*
| Value | Description |
| :---- | :---- |
| **Universal** | Applies to all; no exceptions based on identity or circumstance |
| **General** | Applies broadly with limited exceptions; default applicability |
| **Contextual** | Applies within specific contexts, domains, or situations |
| **Particular** | Applies to specific individuals, groups, or cases |
| **Personal** | Applies only to the individual holding the value; purely subjective |

### 4.2.2 Dimension 2: Weight
*How important the value is relative to others.*
| Value | Description |
| :---- | :---- |
| **Absolute** | Cannot be outweighed; lexically prior to other values; no trade-offs permitted |
| **Dominant** | Very high weight; outweighs most other values in most circumstances |
| **Significant** | Substantial weight; important factor in deliberation |
| **Moderate** | Some weight; contributes to overall evaluation but easily outweighed |
| **Minimal** | Little weight; matters only marginally in most contexts |

### 4.2.3 Dimension 3: Conditionality
*Under what conditions the value applies.*
| Value | Description |
| :---- | :---- |
| **Unconditional** | Applies regardless of circumstances; no conditions for applicability |
| **Prima Facie** | Applies unless overridden by competing considerations; defeasible |
| **Conditional** | Applies only when specific conditions are met |
| **Contingent** | Applicability depends on variable circumstances; highly context-sensitive |
| **Hypothetical** | Applies only given certain goals or commitments; instrumental conditionality |

### 4.2.4 Dimension 4: Source
*Where the value comes from.*
| Value | Description |
| :---- | :---- |
| **Objective** | Exists independently of any valuer; mind-independent value |
| **Intersubjective** | Emerges from shared practices, agreements, or collective recognition |
| **Subjective** | Depends on individual attitudes, preferences, or responses |
| **Constructed** | Created through deliberate human activity; made rather than found |
| **Emergent** | Arises from interaction without being reducible to its sources |

### 4.2.5 Dimension 5: Comparability
*Whether the value can be compared or traded off against others.*
| Value | Description |
| :---- | :---- |
| **Commensurable** | Can be measured on a common scale with other values; fully comparable |
| **Comparable** | Can be ranked or ordered relative to other values without common measure |
| **Roughly Comparable** | Can be compared in broad terms but not precisely ranked |
| **Incomparable** | Cannot be ranked against certain other values; no ordering possible |
| **Incommensurable** | Cannot share a common measure; trade-offs are impossible or inappropriate |

# 5. Dynamics
*How values interact and change over time.*

## 5.1 Dynamics Table (mandatory)
| Dynamic | Description |
| :---- | :---- |
| **Conflict** | Values coming into tension; pursuing one sacrifices another |
| **Reinforcement** | Values supporting and strengthening each other; synergy |
| **Hierarchy** | Values ordered in relations of priority; some higher than others |
| **Transformation** | Values changing over time in weight, scope, or content |
| **Realisation** | Values being achieved or instantiated; the gap between valued and actual |

## 5.2 Dynamic Notes (recommended)
- **Conflict:** Tensions where advancing one value undermines another (e.g., freedom vs security; efficiency vs equity; individual wellbeing vs ecological integrity).
- **Reinforcement:** Synergies where advancing one value strengthens another (e.g., truth and understanding; justice and dignity; community and belonging).
- **Hierarchy:** Priority ordering where some values constrain or outrank others (e.g., absolute constraints; ends over means; sacred values over instrumental ones).
- **Transformation:** Drift or change in meaning, weight, or scope across time, context, and culture (e.g., privacy valued less in some digital contexts; ecological value rising across generations).
- **Realisation:** The gap between what is valued and what is actual, and the practical work required to instantiate the value (e.g., freedom valued but constrained by insecurity; justice aspired to but delivery obstructed).

# 6. Interface Types
*Where values connect to reasoning, judgment, and action.*

## 6.1 Interface Types Table (mandatory)
| Type | Name | Description |
| :---- | :---- | :---- |
| **Type A** | **Justification Interfaces** | Where values provide reasons for action or belief. |
| **Type B** | **Evaluation Interfaces** | Where values provide standards for assessment and judgement. |
| **Type C** | **Motivation Interfaces** | Where values connect to desire, commitment, and action. |

## 6.2 Interface Prompts (recommended)
- **Type A (Justification):** What reasons does the value generate? For what actions or beliefs? With what strength? What would override these reasons?
- **Type B (Evaluation):** What standards does the value impose? What counts as success/failure/excellence? Who judges and by what evidence? What metrics capture this value?
- **Type C (Motivation):** Does recognising the value move people to act? What enables or blocks motivation? What incentives, power relations, or identity factors shape whether people pursue this value in practice?

# 8. Hiddens Source Analysis & Cross-Check (mandatory)

## 8.1 Hiddens Source Analysis (framework-specific)
| # | Generator | Pattern | Remedy |
| :---- | :---- | :---- | :---- |
| **1** | **Value Blindness** | Failing to see operative values; implicit values never named or examined | Deliberate value-seeking through interviews, observation, incentive analysis |
| **2** | **Value Monism** | Reducing all values to one type (profit, efficiency, safety, growth); single-metric domination | Attention to pluralism and incommensurability; plural criteria governance |
| **3** | **Category Neglect** | Ignoring whole categories of values (aesthetic, ecological, social); narrow team composition | Systematic scan across all six categories; rotate perspectives; add domain experts |
| **4** | **Fixity Assumption** | Treating dynamic values as fixed; outdated codes/missions; ignoring cultural drift and generational change | Attention to value dynamics and change; periodic re-elicitation; transformation scenarios |

## 8.2 Hiddens Six-Category Scan (Tier 1 – invariant)
| Hiddens meta-category (I–VI) | Why it may be active in value analysis | Typical symptoms | Default checks to run |
|---|---|---|---|
| I  Perceptual | Values are often implicit and attention-mediated; salience drives what is "seen" as important. | "We never discussed values"; surprise moral/aesthetic backlash; stakeholders talk past each other; "nobody mentioned that matters". | Elicit implicit values (interviews, narrative probes, behaviour observation); "what would be unacceptable?" prompts; fresh-eyes review across roles; executive vs frontline divergence. |
| II Systemic | Methods (metrics, dashboards, economic models, org structures) encode value assumptions; optimisation hides normative choices. | KPI theatre; "the model says" replacing deliberation; single-objective optimisation; metrics driving perverse outcomes; incentive misalignment with stated values. | Audit objective functions and proxies; map what is excluded by design; require plural criteria and non-commensurable constraints; incentive-value alignment check. |
| III Informational | Values travel through institutions and communications; reporting filters what counts as "important". | Sanitised briefings; suppressed dissent; selective stakeholder voice; "that issue never reaches leadership"; value concern disappears in reporting. | Access/provenance checks on value claims; source-independence check; include marginalised perspectives; protected truth channels; channel audit for value filtering. |
| IV Temporal | Values drift; historical sedimentation shapes what is taken for granted as "normal"; stale mission statements; generational shifts undetected. | Stale value statements; sudden generational conflicts; "we've always done it this way"; repeated "why are we doing this?" questions. | Historical baseline review; track value drift indicators; scenario stress tests for value shifts; time-horizon alignment check; re-elicitation cadence. |
| V  Relational | Position, power, and identity shape what can be valued and voiced. | Executive vs frontline misalignment; legitimacy disputes; positional silencing ("people don't speak up about what they value"); identity-protective value rigidity. | Multi-position inquiry; power/incentive mapping; participatory workshops; disagreement log as data; psychological safety audit; voice protection mechanisms. |
| VI Ontological | Sometimes the category set is inadequate; new forms of value emerge (or old ones become salient). | "None of the categories fit"; persistent normative anomalies; repeated exceptions and special cases; value disputes that seem to elude standard frames. | Category-challenge session; propose new value type (Section 3.7); test mapping across dimensions/dynamics/interfaces; involve affected parties in definition. |

## 8.3 Hiddens Crosswalk (Tier 2 – default mapping layer)
| Values hiddens source | Likely Hiddens meta-categories implicated | What to look for (diagnostic cues) | Default mitigation move |
|---|---|---|---|
| Value Blindness | I Perceptual; III Informational; V Relational | Implicit values never surfaced; stakeholder values absent from artefacts; silence or fear around "what matters"; values only visible through contradiction/backlash. | Run structured value elicitation (interviews, sorting exercises, hypothetical scenarios); expand stakeholder set; require explicit value register with provenance notes; protect space for naming uncomfortable values. |
| Value Monism | II Systemic; VI Ontological | Single-number optimisation; one metric dominates; "all values reduce to X" rhetoric; commensuration overreach; model-driven overconfidence. | Enforce plural criteria in governance; separate non-tradeable constraints; mark incommensurables explicitly; require value trade-off rationale statements; audit incentives for value reductionism. |
| Category Neglect | II Systemic; V Relational; VI Ontological | Persistent omission of a meta-category (e.g., aesthetics ignored; ecological values absent; social values traded away); narrow team composition; role-based blinders; "soft values" dismissed. | Mandatory six-category scan; rotate perspectives; add domain experts and affected parties; challenge inherited exclusions; track which categories are consistently devalued. |
| Fixity Assumption | IV Temporal; II Systemic | Treating values as permanent; ignoring cultural drift; outdated codes/policies; "frozen" mission statements; generational shifts missed; no re-elicitation rhythm. | Track value drift; add periodic re-elicitation cadence (quarterly minimum); include transformation scenarios; monitor generational/market/regulatory shifts; flag when values haven't been revisited. |

## 8.4 Embedded Hiddens Micro-Protocol (standard prompts)
1. Run the Tier 1 scan across all six Hiddens meta-categories (assume hiddens exist).
2. For shortlisted hiddens, rate mechanism (Value Blindness/Monism/Category Neglect/Fixity), manifestation (perceptual/systemic/informational/temporal/relational/ontological), reducibility, detectability, agency, consequence.
3. Assign at least one detection test and one remediation lever per high-consequence hidden; map interaction chains.
4. Run a hiddens source analysis and record unresolved unknowns.
5. Map Tier 1 findings to Tier 2 mitigation moves.

## 8.5 Escalation Rule (Tier 3 – full Hiddens run)
Escalate to a full 30-type scan + detection/remediation matrix mapping if any apply:
- High consequence (safety, regulatory, existential, severe harm potential to sacred values)
- Adversarial context / strategic value concealment
- Persistent disagreement or inconsistent evidence on what is valued
- Repeat failures despite prior value-conscious interventions
- Strong suspicion of cascades/reinforcement across hidden types (e.g., Value Monism driving Category Neglect driving Temporal drift)
- Value conflict touches core legitimacy or identity

- Full value register and stakeholder provenance log

- Review cadence (default): re-run Value Identification + Category Scan quarterly for fast-changing contexts; at least annually for stable contexts; immediately after crises, major organisational changes, legitimacy challenges, or repeat failures

# 10. Framework Principles (mandatory)

## 10.1 Principles Table
| Principle name | Description |
|---|---|
| 1. Value Realism | Treat values as real and outcome-shaping (even when contested). Value errors are practical errors, not matters of opinion. Values matter for what gets done and why it fails. |
| 2. Value Pluralism | Assume multiple irreducible value kinds; avoid collapsing ethics, epistemics, aesthetics, practical goods, and social goods into one scale. Sacred values and incommensurables are real constraints. |
| 3. Value Embeddedness | Assume all analysis is never value-neutral; make value presuppositions explicit and examinable. This framework itself rests on values (pluralism, transparency, deliberation). |
| 4. Value Dynamics | Treat values as interacting and time-varying; map conflict, reinforcement, hierarchy, transformation, and realisation gaps. Values drift. Periodic re-elicitation is not optional. |
| 5. Reflexive Application | Apply the framework to its own assumptions and outputs. What values does the analysis prioritise, and why? Whose values are represented in the framework's design? |
