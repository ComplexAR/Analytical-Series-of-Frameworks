## Section 2: Operational Definition

The Learning and Adaptation framework classifies learning mechanisms and adaptation pathways through six meta-categories and thirty core types, profiles learning across five dimensions (experiential basis, codification, transfer, integration, institutional embedding), maps ten dynamics of how learning succeeds or fails and how organisations drift or adapt, and surfaces learning-specific visibility failures via tiered Hiddens scans. It enables feedback loop design, outcome attribution, lesson extraction and transfer, adaptive decision-making, and institutional learning that accumulates knowledge and reduces repeat failure. It supports both individual learning (personal knowledge development) and organisational learning (institution-wide knowledge capture and change).

## Section 5: Condensed Dynamics

| Dynamic pattern | Signature | Mechanism | Consequence |
|---|---|---|---|
| Lesson extraction failure | Failure occurs; no structured debrief; lessons remain tacit; learning decays; same failure repeats | Debrief culture absent; time pressure to move forward; blame dynamics preventing honest analysis; incentive to forget failures | Repeat failure cycles; institutional learning blocked; credibility eroded; unnecessary recurrence suffering |
| Causal misattribution in learning | Failure attributed to proximal cause not root cause; lesson focuses on wrong mechanism; similar root causes produce new failures | Hindsight bias; incentive to protect system rather than learn; insufficient causal analysis; expertise gaps | Lessons ineffective at preventing similar failures; learning failure creates false confidence in remediation; brittleness persists |
| Single-loop learning trap | Organisation responds to failure but addresses symptom not root cause; problem recurs; cycle repeats at higher cost | Incentive pressure for quick fix; systemic root cause invisibility; insufficient causal analysis; sunk costs in incremental fixes | Chronic problem persists; repeat failures; escalating cost; organisational frustration; adaptation ceiling hits |
| Knowledge transfer failure | Knowledge created but not transferred; created in one team but duplicated in another; expertise concentrated; tacit knowledge lost | Knowledge holder turnover; poor transfer mechanisms; incentive misalignment (knowledge retention advantage); isolation | Knowledge fragmentation; duplicate learning; lost opportunity for reuse; capacity utilisation suboptimal; organisational memory loss |
| Feedback loop delay and responsiveness loss | Learning feedback exists but lags are long; decision-to-feedback timescale prevents rapid adaptation; slow learning | Monitoring delay; causal attribution lag; decision cycle mismatches feedback cycle; institutional inertia | Adaptation delayed beyond decision windows; course correction impossible; learning too late to matter; brittleness during lag |
| Defensive reasoning and learning avoidance | Honest analysis threatens identity or authority; learning becomes defensive; uncomfortable truths suppressed; false consensus replaces learning | Identity stakes in decisions; power protection; retaliation risk for raising difficult issues; psychological safety low | Learning blocked by defensiveness; organisation unable to acknowledge problems; brittleness from unaddressed issues; external pressure required to change |
| Best-practice transfer failure | Practice successful in original context; transferred to new context without adaptation; context differences not recognised | Confidence in transferability; boundary-condition blindness; insufficient local customisation; path dependency of success | Transfer fails; local adaptation delayed; false confidence from prior success masks context sensitivity; learning transfer incomplete |
| Institutional forgetting | Learning documented but forgotten; knowledge repository created but not used; institutional memory gaps appear cyclically | Documentation without discovery mechanism; knowledge about knowledge lost; institutional inertia towards alternatives; cost pressure | Repeated reinvention of wheel; lost institutional knowledge; duplicate learning across generations; knowledge credibility questioned |
| Adaptation rigidity and path dependence | Organisation learns and adapts; process becomes institutionalised; rigidity increases; pre-adaptation flexibility is lost | Institutionalisation locks in adaptation; sunk costs prevent reversal; path dependency increases; incentive to maintain past decisions | Organisation becomes brittle to new challenges; fails when context shifts; prior adaptation becomes liability; learning loop closes |
| Organisational learning fatigue and burnout | Many learning initiatives; change fatigue emerges; staff disengage from learning; learning culture erodes | Over-initiation without completion; change cascade without breathing room; learning demands exceed capacity; incentive fatigue | Learning initiatives fail from engagement loss; cynicism about change; staff resistance; institutional learning capability degrades |

## Section 6: Compact Hiddens Cross-Check Prompts

### 6a. Scan Questions by Meta-Category (I–VI)

| Hiddens meta-category | Scan question |
|---|---|
| I (Perceptual) | What learning is not being extracted because failure analysis is insufficient; what lessons are invisible due to inattention? |
| II (Systemic) | How do institutional structures enable or prevent learning; what structural barriers to learning embedding exist? |
| III (Informational) | What knowledge is created but not transferred; what learning repository exists but is inaccessible; what tacit knowledge is lost? |
| IV (Temporal) | What feedback loops are delayed; what trend learning is not happening because monitoring lags are too long; what feedback cycles miss decision windows? |
| V (Relational) | Who knows lessons from failure but remains silent due to retaliation risk; what dissent about what to learn is suppressed? |
| VI (Ontological) | What learning paradigm forecloses alternative interpretation of failure; what is treated as the only lesson to extract? |

### 6b. Primary Hiding Mechanisms

- Lesson extraction failure (insufficient analysis; learning remains tacit)
- Causal misattribution in learning (root-cause invisible; lesson targets wrong mechanism)
- Single-loop learning trap (symptom addressed, root persists; problem recurs)
- Knowledge transfer failure (knowledge created but not shared; duplication and loss)
- Defensive reasoning (uncomfortable truths suppressed; learning blocked by identity threat)
- Feedback loop delay (learning delayed beyond decision window; adaptation impossible)
- Institutional forgetting (knowledge documented but inaccessible; forgotten and reinvented)
- Adaptation rigidity (learning institutionalised as rigid procedure; flexibility lost)

### 6c. Detection/Remediation Interface Pointers

| When this type of Hidden emerges | Invoke this framework | Interface type | Action |
|---|---|---|---|
| Lesson extraction failure | Framework of Learning & Adaptation (debrief design); Framework of Diagnosis | Structured debriefing; root-cause analysis; documented lessons | Implement systematic debrief protocol; strengthen causal analysis before extracting lessons |
| Knowledge transfer failing | Framework of Knowledge; Framework of Learning & Adaptation | Knowledge-transfer mechanism design; documentation and discovery system | Design knowledge-transfer pathways; improve knowledge accessibility; reduce knowledge concentration |
| Causal misattribution in learning | Framework of Diagnosis; Framework of Learning & Adaptation | Competing-hypothesis testing; causal-analysis rigor; root-cause verification | Conduct rigorous causal analysis; test alternative hypotheses; verify root-cause before implementing lessons |
| Defensive reasoning blocking learning | Framework of Culture & Norms; Framework of Learning & Adaptation | Psychological safety assessment; blame-culture reduction; honest-analysis incentives | Reduce blame culture; protect honest analysis; increase psychological safety; reward candour about problems |
| Feedback loop delay | Framework of Learning & Adaptation (feedback redesign); Framework of Metrics | Feedback-cycle analysis; monitoring redesign; faster-feedback mechanisms | Accelerate feedback loops; increase monitoring frequency for critical parameters; reduce lag |

## Section 7: LLM Execution Protocol

### 7a. Light Depth Execution (Scan-level)

1. Identify a recent failure, near-miss, or decision outcome; assess whether systematic lesson extraction occurred.
2. Map feedback loop for this outcome: What was observed? How quickly was it understood? How quickly did understanding feed back to decision-making?
3. Run Tier 1 Hiddens scan: check for lesson extraction failure, causal misattribution, defensive reasoning blocking learning, feedback loop delay.
4. Identify whether learning from this outcome was transferred to others or remained individual/isolated.
5. Flag any single-loop learning patterns (symptom addressed, root cause invisible) or feedback loop delays for escalation to governance redesign.

### 7b. Full Depth Execution (Complete framework run)

1. Conduct learning audit: identify significant failures, near-misses, and successes; assess whether systematic learning extraction occurred for each.
2. Classify learning types (meta-categories I–VI); profile each across five dimensions (experiential basis, codification, transfer, integration, institutional embedding).
3. Map learning feedback loops: for each outcome, trace from observation through understanding to decision-making adaptation; measure loop timeliness.
4. Assess knowledge transfer: what learning is created; how is it transferred across organisation; what is lost or duplicated?
5. Evaluate institutional learning: are lessons embedded in procedures, structures, incentives, culture; or do they remain individual/transient?
6. Run Tier 1 + Tier 2 Hiddens scan: for each high-consequence learning gap, identify detection interface (how to surface learning failure) and remediation (how to enable learning).
7. Identify learning-system design priorities: debrief protocols, causal-analysis rigor, knowledge-transfer mechanisms, feedback-loop acceleration, institutional-memory systems.

### 7c. Register Updates and Gate Question

| Register | Update type | Trigger |
|---|---|---|
| Learning Backlog | Document significant learning opportunities (failures, near-misses, successes); identify lesson extraction, transfer, and institutional-embedding needs | From outcome analysis |
| Knowledge Register | Record learning-generated knowledge; track transfer pathways; identify tacit knowledge at risk of loss | From learning audits |
| Governance Documentation | Specify debrief protocols; causal-analysis requirements; knowledge-transfer mechanisms; feedback-cycle timelines; institutional-memory systems | For organisational learning infrastructure |

**Gate Question (pre-closure check):** Have we audited learning infrastructure (debrief protocols, causal analysis, knowledge transfer, feedback loops), identified high-consequence learning gaps, run Tier 1 Hiddens scan for lesson-extraction failure and causal misattribution, and designed organisational learning systems for rapid feedback and institutional memory?
