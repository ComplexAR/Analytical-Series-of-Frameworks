## Section 2: Operational Definition

The Agents framework identifies who or what acts within a scenario; classifies actors into six meta-categories and thirty core types; profiles each agent across five key dimensions (Autonomy, Intentionality, Consciousness, Accountability, Temporal Persistence); maps agent interaction patterns (Cooperation, Competition, Domination, Delegation, Emergence); and surfaces hidden agents, masked principals, and distorted agency that false accountability narratives conceal. It populates the Hiddens Register, Evidence Ledger, and Hypothesis/Test Backlog with agent-level blindnesses and governance requirements, enabling downstream frameworks to design accountability structures aligned with actual agency rather than formal roles.

## Section 5: Condensed Dynamics

| Dynamic pattern | Signature (what you observe) | Mechanism (why it happens) | Consequence |
|---|---|---|---|
| Cooperation | Agents coordinate toward shared goals; joint projects, coalitions, alliances, mutual aid visible. | Agents align incentives; shared understanding enforced; positive-sum opportunity recognised. | Creates shared capability and enterprise; also concentrates failure modes (both fail if shared assumption collapses); reduces accountability clarity. |
| Competition | Agents pursue incompatible goals; arms races, strategic adaptation, selection pressures visible. | Scarce resources or incompatible objectives; agents anticipate and counter each other. | Drives innovation (selection pressure) or destructive escalation; amplifies distortion/secrecy; produces winner-take-all concentration. |
| Domination | Asymmetric control: one agent constrains or compels others via authority, coercion, resources, dependency. | Power differentials; ability to force compliance or restrict options; structural inequality present. | Explains compliance and suppression; dominated agents may resist covertly; reveals hidden vulnerability and fragility (system depends on domination continuation). |
| Delegation | Principal–agent relations: one agent acts on behalf of another under mandates, incentives, accountability. | Separation of principal's goals from agent's incentives; information asymmetries; monitoring/enforcement challenges. | Explains accountability gaps, misaligned incentives, automation blame; central to understanding control design; layers of delegation obscure true decision-maker. |
| Emergence | System-level patterns arise from interactions without central control; macro patterns shape micro agent behaviour. | Collective patterns from individual interactions; no agent intends macro outcome; feedback loops reinforce pattern. | Explains unintended outcomes; requires multi-level analysis; can produce robustness or brittleness; reification risk (pattern treated as agent intent). |

## Section 6: Compact Hiddens Cross-Check Prompts

### 6a. Scan Questions by Meta-Category (I–VI)

| Hiddens meta-category | Scan question (1 sentence max) |
|---|---|
| I (Perceptual) | Which actors are missing from the map, especially intermediaries, marginal stakeholders, and agents at interfaces? |
| II (Systemic) | Which agents are wrongly blamed (scapegoating, automation blame) or wrongly exonerated because role-based appearance masks true power? |
| III (Informational) | Where is information about who decided withheld, fragmented across systems, or conflicting across different sources and witnesses? |
| IV (Temporal) | How do identity shifts (rebranding, turnover, succession, model updates) allow accountability to escape or widen gaps? |
| V (Behavioral) | Where do agents conceal actual goals, expertise, constraints; where is espoused-vs-operative behaviour masking true intentionality? |
| VI (Structural) | Where do outcomes emerge from distributed interaction in ways no single actor intends; how does macro pattern conceal individual agency distribution? |

### 6b. Primary Hiding Mechanisms This Framework Detects

- Agent Misidentification (wrongly identifying who acts, overlooking key actors, false attribution)
- Capacity Mismatch (over/underestimating agent capacities; confusing role-prescribed capacity with actual capability)
- Category Neglect (ignoring whole agent categories; defaulting to Individual/Institutional while missing Collective/Distributed/Abstract agents)
- Interaction Blindness (missing how agents interact; treating agents as isolated rather than coupled, delegating, competing, emerging)
- Principal–Agent Concealment (hidden principals, masked intermediaries, true decision-makers invisible behind delegated appearance)
- Accountability Evasion (identity shifts allowing escape; responsibility reassigned to low-autonomy agents; distributed accountability masking true loci)

### 6c. Detection/Remediation Interface Pointers

| When this type of Hidden emerges | Invoke this framework | Interface type | Action |
|---|---|---|---|
| Scapegoating or automation blame | Framework of Causation; Framework of Responsibility | Decision attribution; accountability mapping | Trace decisions from outcome back through delegation chains; identify true locus of control; test if blamed agent had autonomy to choose differently |
| Hidden principals or intermediaries | Framework of Systems; Framework of Causation | Relationship mapping; information/action flow analysis | Map decision pathways; trace information flow; interview margin actors and intermediaries; compare formal structure to actual practice |
| Principal–agent misalignment | Framework of Incentives; Framework of Governance | Incentive audit; principal–agent test | Compare principal's stated goals to agent's observed behaviour; audit what agent is actually rewarded/punished for; test alignment with evidence |
| Distributed agency misidentified as unified control | Framework of Systems; Framework of Hiddens (Tier 2) | System dynamics modelling; emergence detection | Model system interactions; test whether individual agents intended observed outcome; identify tipping points and cascade vulnerabilities |
| Identity shift enabling accountability escape | Framework of Responsibility; Framework of Governance | Identity/succession tracking; responsibility continuity check | Track agent identity across time/changes; map succession and responsibility pass-through; audit whether accountability follows identity |

## Section 7: LLM Execution Protocol

### 7a. Light Depth Execution (Scan-level)

1. List all focal actors mentioned in scenario, evidence, interviews; brainstorm any missing obvious candidates; note obvious gaps (all categories represented?).
2. For each actor, assign to one of six meta-categories (I–VI) and specify one or two core types (1–30); note uncertainty.
3. For top actors (3–5), quickly estimate three key dimensions: Autonomy (High/Medium/Low), Intentionality (Yes/Functional/No), Accountability (Clear/Partial/Distributed).
4. Identify which one or two interaction patterns (Cooperation/Competition/Domination/Delegation/Emergence) dominate; note any hidden interactions.
5. Run Tier 1 Hiddens scan across six meta-categories; shortlist top 2–3 invisible agents or distorted agency patterns; flag for Tier 2 if high-consequence.

### 7b. Full Depth Execution (Complete framework run)

1. Create comprehensive actor/agent inventory: all individuals, collectives, institutions, artificial systems, distributed agents, and abstract agents in scope; note versions, deployment scale, user populations.
2. Classify each agent into meta-category (I–VI) and core type (1–30); document classification rationale; note any taxonomy refinements or domain-specific deviations.
3. Profile each top agent across five dimensions: Autonomy (range: Full to Minimal), Intentionality (Full Conscious to Non-Intentional), Consciousness (where relevant and contested), Accountability (Full to None), Temporal Persistence (Continuous to Momentary).
4. Map all five interaction patterns (Cooperation, Competition, Domination, Delegation, Emergence); identify which are active, nascent, or high-risk; trace cascades and feedback loops.
5. For each focal agent, assess three interface types: Communication Interfaces (signal fidelity, incentives, withholding), Action Interfaces (coupling, externalities, collision risk), Constitution Interfaces (what constitutes agent capacity and identity).
6. Run Tier 1 + Tier 2 Hiddens scan: for each high-relevance Hidden (Agent Misidentification, Capacity Mismatch, Category Neglect, Interaction Blindness, Principal–Agent Concealment, Accountability Evasion), identify mechanism, detectability, consequence; assign detection and remediation interfaces.
7. Document residual Unknowns about agent identity, autonomy, accountability; apply Escalation Rule; generate Residual Unknowns register, closure artefacts, and monitoring plan.

### 7c. Register Updates and Gate Question

| Register | Update type | Trigger |
|---|---|---|
| Hiddens Register | Add shortlist of agent-specific Hiddens with meta-category tag (I–VI), mechanism, detectability, consequence, suspected locus. | Tier 1 scan completion; escalate any high-consequence/high-uncertainty Hidden to Tier 2. |
| Evidence Ledger | Record agent classifications, dimensional assessments, interaction pattern presence/absence, evidence quality, provenance. | After agent classification and dimensional profiling; note gaps (e.g., capacity assessment, autonomy testing, accountability proof). |
| Hypothesis/Test Backlog | Add hypotheses about hidden agents, principal–agent alignment, autonomy distribution, delegation chains; propose tests (interviews, role-based evidence comparison, incentive audits). | Identified Unknowns and residual Hiddens; escalation decisions. |
| Information Requests | Request actor lists, org charts, decision logs, interview transcripts, email trails, role definitions, incentive structures, succession records. | To reduce Unknowns and validate Hiddens detection; coordinate with Evidence and Responsibility frameworks. |

**Gate Question (pre-closure check):** Have we identified all focal agents across six meta-categories, assessed their key dimensions (Autonomy, Intentionality, Accountability), mapped their interactions, run a Tier 1 Hiddens scan, and escalated any high-consequence agent-level Unknowns to Tier 2/3 or downstream frameworks (Responsibility, Causation, Governance)?
